# Network Guard - 内网设备监控守卫

[![AstrBot](https://img.shields.io/badge/AstrBot-v4.16+-blue)](https://github.com/Soulter/AstrBot)
[![License](https://img.shields.io/badge/License-MIT-green)](LICENSE)

## 📖 简介

内网设备监控守卫是一个 AstrBot 插件，用于监控局域网设备接入情况。发现新设备时自动向管理员推送通知，支持白名单管理、设备备注、**黑名单自动攻击**（黑名单设备上线即自动断网）。

**适用场景：** 防止蹭网、监控未知设备接入、家庭网络安全管理。

---

## ✨ 功能

| 功能 | 说明 |
|------|------|
| 📡 **自动扫描** | 宿主机每分钟收集 ARP 信息，实时更新设备列表 |
| 🔔 **新设备通知** | 发现未记录过的设备时自动推送消息到 QQ |
| ✅ **白名单管理** | 信任的设备不再被视为陌生的 |
| 🏷️ **设备备注** | 给设备起名，`守卫列表` 直接显示名称 |
| ⛔ **黑名单** | 拉黑设备后，上线即自动 ARP 攻击断网 |
| ⚔️ **手动攻击** | 对指定 IP 发起单播 ARP 攻击（不影响其他设备） |
| 📋 **设备列表** | 查看局域网所有在线设备及备注名称 |

---

## 📟 指令列表

| 指令 | 说明 |
|------|------|
| `守卫扫描` | 立即扫描局域网，列出所有在线设备 |
| `守卫列表` | 查看已记录设备（含备注名称） |
| `守卫备注 <IP/MAC> <名称>` | 给设备起名，绑定到 MAC 地址 |
| `守卫信任 <MAC> <名称>` | 加入白名单（等同于备注） |
| `守卫移除 <MAC>` | 从白名单移除 |
| `守卫拉黑 <IP>` | 加入黑名单（上线自动攻击） |
| `守卫赦免 <IP>` | 移出黑名单，停止攻击 |
| `守卫黑名单` | 查看黑名单列表 |
| `守卫攻击 <IP> [秒数]` | 手动 ARP 攻击（单播，默认 60 秒） |
| `守卫停止 <IP>` | 恢复被攻击设备的网络 |
| `守卫测试通知` | 发送测试推送通知 |
| `守卫帮助` | 显示帮助信息 |

### 指令示例

```
守卫扫描
守卫列表
守卫备注 192.168.31.28 蹭网老王
守卫拉黑 192.168.31.105
守卫攻击 192.168.31.105 120
守卫赦免 192.168.31.105
```

> **注意：** 发送指令不需加 `/` 前缀。

## ⚙️ WebUI 配置

在 AstrBot WebUI → 插件配置 → `network_guard` 中可设置：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `notify_session` | `野生t:FriendMessage:814487409` | 通知推送目标会话 |
| `scan_interval` | `10` | 自动扫描间隔（分钟） |
| `notify_on_new` | `true` | 新设备通知开关 |
| `ssh_host` | `192.168.31.42` | 宿主机 IP（用于 ARP 攻击） |
| `ssh_user` | `root` | 宿主机 SSH 用户名 |
| `ssh_password` | — | 宿主机 SSH 密码 |
| `known_devices` | `[]` | 白名单（格式: `MAC:名称`） |
| `blacklist` | `[]` | 黑名单 MAC 地址列表 |

## 🔧 工作原理

```
宿主机（飞牛/NAS）
  └─ cron（每1分钟）
       └─ ip neigh show → 写入共享文件
                          ↓
AstrBot 容器
  └─ network_guard 插件
       ├─ 读取共享文件 → 扫描设备列表
       ├─ 比对白名单/黑名单 → 推送通知 / 自动攻击
       └─ arpspoof（通过 SSH）→ 单播 ARP 攻击
```

---

## ⚠️ 注意事项

- ARP 攻击功能仅适用于**你自己的局域网**，请勿用于非法用途
- 攻击使用 `arpspoof` 工具（单播），仅影响目标设备，不会波及其他设备
- 黑名单设备上线后持续攻击直到离线，离线后自动停止
- 非 Docker 环境无需 SSH 配置，自动使用本地命令
